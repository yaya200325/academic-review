# Course Paper Mode

## Target parameters
- Reference count: 15–30 papers (or user's specified count, whichever applies)
- Word count: 3000–6000 words for main body (or user's specified count)
- Main sections: 4–6

## Quality targets
- Accurate synthesis, not exhaustive coverage
- Clear logical flow: background → current state → key methods → open gaps
- Each section opens with a thesis sentence
- Horizontal comparison preferred over paper-by-paper listing

## Depth calibration for synthesis notes
Focus on: `core_claims`, `method_groups`, `consensus_points`, `evidence_gaps`
You may leave `controversies` brief if the topic is not heavily debated.

## Diagram guidance
- Generate 1–2 diagrams if they help explain the topic (e.g., a conceptual taxonomy or a method flow)
- Do not generate diagrams for every section — only where they add explanatory value

## Table guidance
- Generate a comparison table only if there are 3+ distinct methods being compared in the same section
- One table is usually sufficient for a course paper

## Tone
Clear, informative, moderately formal. Does not need to be as comprehensive as a journal review.
